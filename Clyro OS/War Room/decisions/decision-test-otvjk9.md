# Décision : Test

## Contexte
### Marché : Plateformes de test en ligne (logiciel, éducation, certification)

#### **Principaux Concurrents**
1. **Pour les tests techniques (développeurs)** :
   - HackerRank
   - Codility
   - TestGorilla
   - CoderPad
2. **Pour l'évaluation cognitive & comportementale** :
   - AssessFirst
   - Central Test
   - Pymetrics
3. **Pour les certifications professionnelles** :
   - Pearson VUE
   - Certiport

#### **Offres & Pricing**
- **HackerRank** : Tests de code, évaluations techniques. Pricing : sur devis, segmenté (start-up à grande entreprise).
- **Codility** : Évaluations de programmation automatisées. Pricing : abonnements à partir de ~500$/mois.
- **TestGorilla** : Tests pré-embauche (techniques, soft skills). Pricing : ~30$/candidat/mois pour les forfaits de base.
- **AssessFirst** : Tests de personnalité et de motivation. Pricing : ~50€/candidat.
- **Pearson VUE** : Tests de certification surveillés en centre ou en ligne. Pricing : variable par certification.

#### **Angles de Différenciation**
1. **Expérience candidat** : Interface utilisateur intuitive, feedback détaillé post-test.
2. **Personnalisation** : Création de tests sur mesure vs. catalogue standardisé.
3. **Intégrations** : Connexion fluide avec les ATS (Applicant Tracking Systems) populaires.
4. **Surveillance & intégrité** : Technologies anti-triche (webcam, détection de fraude).
5. **Analytics avancés** : Rapports prédictifs sur la performance et l’ajustement culturel.

#### **Opportunités**
1. **Niche spécialisée** : Tests pour des rôles émergents (IA, cybersécurité, green tech).
2. **Expérience mobile-first** : Optimisation pour candidature via smartphone.
3. **Gamification** : Introduire des éléments de jeu pour augmenter l'engagement.
4. **Partnerships** : Collaborer avec des écoles/universités pour les évaluations en continu.
5. **Focus PME** : Offres simplifiées et abordables pour les petites entreprises.

#### **Menaces**
1. **Concurrence intense** : Nombreux acteurs sur un marché encombré.
2. **Réticence des candidats** : Fatigue liée aux tests multiples, crainte du biais algorithmique.
3. **Évolutions réglementaires** : RGPD, législation sur l’équité des algorithmes d’embauche.
4. **Émergence de l'IA** : Risque de contournement via ChatGPT/outils similaires pour les tests techniques.
5. **Consolidation du marché** : Rachats par de grands groupes (RH, EdTech).

### **Actions Recommandées pour Clyro**
1. **Identifier un segment précis** : Éviter l’affrontement direct avec les leaders. Cibler par exemple les tests pour métiers tech émergents.
2. **Développer un USP technologique** : Investir dans des outils anti-triche innovants ou des analytics prédictifs supérieurs.
3. **Adopter un pricing flexible** : Modèle "freemium" pour attirer les PME, pricing à l’usage complémentaire aux forfaits.
4. **Construire un écosystème** : Intégrations prioritaires avec les ATS les plus utilisés par la clientèle cible.
5. **Communiquer sur l'éthique** : Mettre en avant des tests équitables, transparents et conformes RGPD pour se différencier.
