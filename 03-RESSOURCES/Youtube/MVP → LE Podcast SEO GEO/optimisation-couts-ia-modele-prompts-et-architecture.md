# Optimisation coûts IA : modèle, prompts et architecture

## Idées clés

- Les coûts IA peuvent être divisés par 3-4 avec le bon choix de modèle (ex: DeepSeek V4 vs Anthropic Sonet)
- Optimiser l'input (prompts) peut réduire les coûts de 30-40% via le minimalisme et le nettoyage régulier
- Deux approches pour déployer l'IA en entreprise: usage libre (Excel-like) vs tunnels de travail structurés (SaaS-like)
- Le CLI (abonnement fixe) est optimal pour les individus/small teams, l'API (pay-per-use) pour les applications scale
- Vercel AI Gateway permet de switcher facilement entre modèles sans modifier le code

## Tactiques actionnables

- Tester DeepSeek V4 comme alternative rentable aux modèles Anthropic/Claude
- Utiliser LM Arena pour benchmarker les modèles IA selon son usage spécifique
- Implémenter Vercel AI Gateway pour faciliter le switching entre modèles
- Créer des prompts minimalistes et nettoyer régulièrement les processus obsolètes
- Pour les individus: privilégier les abonnements CLI (Cloud Code Max), pour les applications: API avec auto-topup
- Segmenter les clés API par usage/personne pour mieux tracker la consommation

## Citations

> Chez Chato, on va passer de 10000€ par mois de coût IA à 3000€. Plus vous êtes efficace, moins ça va vous coûter cher

> Le choix du modèle : on peut tabler sur un coût divisé par 3 par 4. La quantité d'input peut réduire de 30-40%

> Les entreprises comme Microsoft et Uber ont brûlé leur budget IA en 4 mois au lieu de 12 car les équipes utilisaient l'IA sans objectif business
